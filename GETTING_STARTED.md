Read this file fully before starting.

## Step-by-Step: How to Start

### 1. Install Prerequisites (one-time, on your machine)

Open a terminal and check each:

**Python 3.11 or higher**
```
python --version
```
Need it? → https://www.python.org/downloads/ (check "Add to PATH" on Windows)

**uv — Python package manager**
```
uv --version
```
Not installed? Run one of:
```bash
# Windows (PowerShell)
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"

# Mac / Linux
curl -LsSf https://astral.sh/uv/install.sh | sh
```
After installing, close and reopen your terminal.

**Git**
```
git --version
```
Need it? → https://git-scm.com/downloads (required to push your project to GitHub)

**Antigravity IDE**
If you don't have it, download and install it from the link provided to you.

---

### 2. Get a Gemini API Key

Go to → https://aistudio.google.com/apikey

Click **Create API Key** → copy it → keep it safe.
You'll paste it into a `.env` file during setup. Never share it.

---

### 3. Create a Workspace Folder

Create a new empty folder anywhere on your computer (e.g., name it `adk-workspace`). 

Do not worry about naming it after your project yet—the AI will automatically scaffold your actual project in a subfolder inside this workspace folder.

---

### 4. Copy the Playbook Into Your Workspace Folder

Copy `agent_builder_playbook.md` and `GETTING_STARTED.md` into the workspace folder you just created.

Your workspace folder should look like:
```
adk-workspace/
├── GETTING_STARTED.md
└── agent_builder_playbook.md   ← just these files for now
```

---

### 5. Open the Workspace Folder in Antigravity IDE

In Antigravity IDE:
- **File → Open Folder** → select your `adk-workspace` folder.

⚠️ Make sure you open the **workspace folder**, not a parent folder or the Desktop.

---

### 6. Start the AI Build

In the Antigravity IDE chat, type exactly:

```
@agent_builder_playbook.md start
```

The AI will read the playbook and begin guiding you through the build automatically.
You don't need to copy-paste any prompts — the AI handles everything.

## Commands You'll Use (the AI runs these for you)

> [!IMPORTANT]
> When running commands manually (e.g. `uv sync`, `make playground`, `make run`, `git init`), you MUST navigate into your project subfolder first:
> ```bash
> cd <project-name>
> ```

| Command | What it does |
|---------|-------------|
| `uvx google-agents-cli setup` | Installs agents-cli + ADK skills |
| `agents-cli info` | Shows installed version and skills |
| `uv sync` | Installs Python dependencies (run inside `<project-name>`) |
| `make playground` | Opens the agent test UI at localhost:18081 (run inside `<project-name>`). **On Windows: See FAQ below.** |
| `make run` | Runs the agent as a local web server (run inside `<project-name>`) |

### How to Run & Stop the Project (AI or Manual)

#### Option A: Ask the AI to run or stop it for you
If your AI assistant is active, simply ask it in the chat:
* 💬 `"Run the playground"` (starts the Dev UI)
* 💬 `"Start the local server"` (starts the FastAPI backend)
* 💬 `"Stop the server"` or `"Stop the playground"` (stops any active background server/playground processes)

#### Option B: Run & stop manually in your terminal
Navigate into your project folder (`cd <project-name>`) and run:

* **To start the Playground UI (http://localhost:18081):**
  * **Windows**:
    ```powershell
    uv run adk web <agent_dir> --host 127.0.0.1 --port 18081 --reload_agents
    ```
    *(Replace `<agent_dir>` with your actual agent source folder name. `agents-cli scaffold` usually names it after your project, NOT `app` — check the subfolder that contains `agent.py`. Using the wrong name causes "no agents found" / "extra arguments" errors.)*
  * **macOS / Linux**:
    ```bash
    make playground
    ```

* **To start the FastAPI Server:**
  * Run:
    ```bash
    make run
    ```

* **To stop the Playground UI or FastAPI Server:**
  * **If running in the foreground**: Press `Ctrl + C` in your terminal window.
  * **If running in the background**:
    * **Windows (PowerShell)**:
      ```powershell
      Get-Process -Id (Get-NetTCPConnection -LocalPort 18081, 8090 -ErrorAction SilentlyContinue).OwningProcess | Stop-Process -Force
      ```
    * **macOS / Linux**:
      ```bash
      lsof -ti:18081,8090 | xargs kill -9
      ```

> [!NOTE]
> **API Quota Protection:** The AI is instructed **not** to run automated browser testing or integration scripts that repeatedly call LLM endpoints, as this exhausts free-tier API quotas. The AI will spin up the server or playground and instruct you on how to test it yourself.

## Frequently Asked Questions

**Q: Do I need a paid Google Cloud account?**
No. Everything uses a free Gemini API key from Google AI Studio.

**Q: Can I describe my own project idea instead of picking from a list?**
Yes. When the AI shows project options, choose "I have my own idea" and describe it.

**Q: What if the AI makes a mistake or I want to change something?**
Just tell it in plain English. e.g., "Change the threshold to $200" or "Add a feedback agent."

**Q: Where is the MCP server? Do I need to install anything for it?**
The AI creates `mcp_server.py` in Phase 3 and adds the MCP SDK to `pyproject.toml`.
No system-level MCP installation needed — it runs as a local process automatically.

**Q: Do I need to deploy to Google Cloud?**
No. Deployment to Google Cloud is **not required**. Everything runs locally on
your machine using `make playground`. Local testing is all you need to build
and demonstrate your agent.

---

**Q: I'm getting a quota or rate limit error from the Gemini API — what do I do?**
Switch to a different model directly in Antigravity IDE:
  1. In Antigravity IDE, click the **model selector** (top of the chat panel)
  2. Pick a different model — e.g. switch from `gemini-2.5-pro` to `gemini-2.5-flash`,
     or try `gemini-2.0-flash` if flash is also throttled
  3. Continue your conversation — the AI will use the new model immediately
  ⚠ Do NOT change the model inside `config.py` — that controls the agent's model,
    not the coding assistant model. Change only the IDE model selector.

---

**Q: The agent itself is running out of API quota during testing — how do I change the model the agent uses?**
By default, the generated project uses `gemini-2.5-flash` (the older `gemini-1.5-*` models are retired and now return 404 errors, so they can't be used). If you hit free-tier limits, `gemini-2.5-flash-lite` has higher daily request limits.
If you want to use a different model for your agent:
  1. Open your `.env` file inside your project folder.
  2. Add or modify the `GEMINI_MODEL` environment variable, e.g.:
     ```
     GEMINI_MODEL=gemini-2.5-flash-lite
     ```
  3. The agent's `config.py` is configured to dynamically read this value and switch the model immediately without any code edits.

---

**Q: I'm on Windows and `make playground` crashes with "Got unexpected extra arguments" — what do I do?**
On Windows, PowerShell/Cmd can automatically expand the `*` wildcard in the internal `--allow_origins '*'` parameter. This causes the command to crash.
To run the playground successfully on Windows, run the command directly from inside your project subfolder:
```powershell
uv run adk web <agent_dir> --host 127.0.0.1 --port 18081 --reload_agents
```
*(Replace `<agent_dir>` with your actual agent source folder — the subfolder containing `agent.py`. It is usually named after your project, NOT `app`. The wrong name causes "no agents found" or "Got unexpected extra arguments" errors.)*
